# Developer Guide

## Dependency Management

Dependencies are managed with [pip-tools](https://pip-tools.readthedocs.io/).

### File roles

| File | Role | Edit manually? |
|---|---|---|
| `requirements.in` | Direct runtime dependencies, loosely specified | Yes |
| `test-requirements.in` | Direct test dependencies (`-r requirements.in` + `pytest`) | Yes |
| `requirements.txt` | Fully pinned runtime lock file (generated) | No |
| `test-requirements.txt` | Fully pinned test lock file (generated) | No |

Never edit the `.txt` files by hand — they are regenerated by `pip-compile` and committed as-is.

### Setup

Install pip-tools into your local virtual environment:

```bash
python -m venv venv
source venv/bin/activate
pip install pip-tools
pip install -r test-requirements.txt
```

### Updating dependencies

**Recompile after editing a `.in` file:**

```bash
pip-compile requirements.in
pip-compile test-requirements.in
```

**Upgrade all packages to the latest compatible versions:**

```bash
pip-compile --upgrade requirements.in
pip-compile --upgrade test-requirements.in
```

**Upgrade a single package:**

```bash
pip-compile --upgrade-package flask requirements.in
```

After recompiling, reinstall and run the tests:

```bash
pip install -r test-requirements.txt
./unittest.sh
```

Then commit both the `.in` and the regenerated `.txt` files together.

### Adding a new dependency

1. Add the package (with optional constraints) to `requirements.in` or `test-requirements.in`:
   ```
   some-package          # latest at compile time
   some-package>=2.0     # minimum version
   some-package>=2.0,<3  # stay on major version 2
   ```
2. Recompile the relevant `.txt` file (see above).
3. Commit both files.

### CI enforcement

The test workflow runs `pip-compile --check` before installing dependencies. This fails the build if the committed `.txt` files are out of sync with the `.in` files, preventing accidental hand-edits to the lock files from going unnoticed.

### Automated updates (Dependabot)

Dependabot is configured to open weekly PRs that bump versions in the `.in` files. After merging a Dependabot PR, run `pip-compile` locally to regenerate the `.txt` files and push them as a follow-up commit.
